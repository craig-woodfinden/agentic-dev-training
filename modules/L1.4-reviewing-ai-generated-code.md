# L1.4 -- Reviewing AI-generated code

**Level:** 1 -- Novice
**Time:** ~60 minutes
**Pair or solo:** Pair recommended -- you'll catch more on someone else's diff than your own.

## You'll be able to

- Run a five-point checklist over any AI-generated diff before you merge it.
- Spot the three failure modes that catch out the most engineers: fabricated APIs, deleted tests, plausible-but-wrong logic.
- Recognise when an AI assistant has quietly expanded the scope of your task.

## Read / watch (15 min)

- Pick **one** PR from your repo's history that was clearly AI-assisted (or do one yourself before this session). You'll come back to it.
- Read your team's existing review guide if there is one. Note: this module *adds* to your normal review process, it doesn't replace it.

If you have time, skim GitHub's [GitHub Copilot best practices](https://docs.github.com/en/copilot/using-github-copilot/best-practices-for-using-github-copilot) for the section on review.

## Do (40 min)

You'll walk a real AI-generated diff through a five-point checklist. The point is to find at least one issue you would have missed before.

**The five-point checklist**

For every AI-generated diff, check these in order. The first three catch the worst bugs; the last two catch the embarrassing ones.

1. **Scope creep.** Does the diff change things you didn't ask for? (Renamed variables outside the scope, added imports, "helpful" reformatting). If yes -- revert anything outside scope and prompt again.
2. **Fabricated APIs.** Pick any unfamiliar function call, library, or config key. Search the docs or grep the repo for it. Does it actually exist? Models hallucinate methods that "feel right".
3. **Plausible-but-wrong logic.** Read the diff out loud, one line at a time. Walk a real input through it on paper. Off-by-one, inverted condition, wrong default -- these read fine and run wrong.
4. **Tests.** Did any tests get deleted, skipped, or weakened? Search for `skip`, `xfail`, `// FIXME`, `pytest.mark.skip` added in this diff. If a test was removed, the reason needs to be explicit.
5. **Failure modes.** What happens on null input, empty list, network error, large input? If the diff has no handling, that's a question for the prompter -- not a blocker, but a question.

**Walk a real diff (30 min)**

Pick the AI-generated PR you identified earlier. Run the checklist over it. Write down every issue you find, including the ones that are fine. Aim for at least one real issue.

Then run the checklist on your own GitHub Copilot-assisted commit from L1.3.

**Save the checklist (5 min)**

Drop the five points into a VS Code snippet, a pinned Teams message, or wherever you'll see it next time. The whole point is that this becomes muscle memory.

## Self-check (5 min)

Make sure you can answer these from memory:

- Recite the five-point checklist.
- Which of the five did you have to look up most? That's your weak spot.
- Name one fabricated API or plausible-but-wrong line you actually found.

## Done when

You can recite the five-point checklist and you've walked it over a real PR. Bonus: you've found at least one issue that the original reviewer missed.
