# L2.4 — Model and cost selection

**Level:** 2 — Practitioner
**Time:** ~75 minutes
**Pair or solo:** Solo build; cohort comparison at the end.

## You'll be able to

- Pick between Haiku, Sonnet, and Opus deliberately rather than by default.
- Estimate the cost of a feature before it ships.
- Apply prompt caching and batching where they actually pay back.

## Read / watch (15 min)

- [Anthropic — Models overview](https://docs.claude.com/en/docs/about-claude/models/overview) — the comparison table.
- [Anthropic — Pricing](https://docs.claude.com/en/docs/about-claude/pricing) — full page.
- [Anthropic — Prompt caching](https://docs.claude.com/en/docs/build-with-claude/prompt-caching) — overview.
- Read the latest Claude model card you can find on docs.claude.com.

## Do (50 min)

You'll run the *same* eval set you built in L2.3 against three models and write a short decision memo.

**Step 1 — Re-run your eval across three models (25 min)**

Modify your runner from L2.3 to take a `model` argument. Run it three times:

```python
for model in ["claude-haiku-4-5", "claude-sonnet-4-6", "claude-opus-4-6"]:
    # ...same loop, capture: pass rate, total tokens in, total tokens out, wall-clock latency
```

Record for each model:
- Pass rate on your eval set.
- Total input + output tokens across all cases.
- Total cost (compute from the pricing page).
- Median per-call latency.

**Step 2 — Write a one-page decision memo (15 min)**

Save as `model_selection.md`. Answer these in plain prose:

- Which model would you pick for this feature, and why?
- What's the cost per 1,000 calls? Per 100,000?
- Where would you switch models — what would have to change for you to upgrade from Haiku to Sonnet, or Sonnet to Opus?
- Where would caching help you? (Long, repeated system prompts are the big one.)

A good memo is a screen of text, not five pages. Make it the thing you'd send your manager when they ask "why are we paying for Opus?".

**Step 3 — Add prompt caching (10 min)**

If your prompt has any long, stable preamble (system prompt with examples, repo description, etc.), add a `cache_control` block:

```python
client.messages.create(
    model="claude-sonnet-4-6",
    system=[
        {"type": "text", "text": LONG_STABLE_SYSTEM_PROMPT, "cache_control": {"type": "ephemeral"}},
    ],
    messages=[{"role": "user", "content": user_input}],
)
```

Run your eval twice in a row. Check `usage.cache_read_input_tokens` on the second run. Recompute cost.

## Discuss (10 min)

Compare with your cohort:

- Did Haiku pass enough cases to be the right pick? People consistently over-estimate how much they need a bigger model.
- Whose memo would you hand to a non-engineer manager?
- What's the cheapest thing in your team's stack right now that's running on too-big a model?

## Done when

You have a one-page decision memo recommending a specific model for a specific feature, with cost figures you can defend.
