# L2.2 — Tool use and function calling

**Level:** 2 — Practitioner
**Time:** ~90 minutes
**Pair or solo:** Solo build; share tool schemas with the cohort at the end.

## You'll be able to

- Design a tool schema the model will actually use correctly.
- Implement the tool-call loop (model → tool → result → model) end-to-end.
- Spot and prevent the three classic failure modes: hallucinated arguments, infinite loops, schema drift.

## Do (60 min)

Build a tiny agent that can answer questions about your team's repos using two tools you define.

**Setup (5 min)**

```bash
pip install anthropic
export ANTHROPIC_API_KEY="..."
```

Pick one repo on disk you'll let the agent search.

**Define two tools (15 min)**

Write JSON-schema definitions for these two tools. Aim for descriptions that someone unfamiliar with your codebase could understand:

```python
TOOLS = [
    {
        "name": "list_files",
        "description": "List files in the given directory of the repo. Use this to discover what code exists before reading it. Returns up to 100 file paths.",
        "input_schema": {
            "type": "object",
            "properties": {
                "path": {"type": "string", "description": "Path relative to repo root. Empty string means repo root."}
            },
            "required": ["path"],
        },
    },
    {
        "name": "read_file",
        "description": "Read the contents of a file in the repo. Use after list_files to inspect code. Files larger than 50KB are truncated.",
        "input_schema": {
            "type": "object",
            "properties": {
                "path": {"type": "string", "description": "Path relative to repo root."}
            },
            "required": ["path"],
        },
    },
]
```

**Implement the loop (30 min)**

```python
import anthropic, os, pathlib

REPO = pathlib.Path("/path/to/your/repo").resolve()
client = anthropic.Anthropic()

def list_files(path):
    p = (REPO / path).resolve()
    if not str(p).startswith(str(REPO)):
        return {"error": "path escapes repo"}   # don't skip this check
    return {"files": [str(f.relative_to(REPO)) for f in p.glob("*")][:100]}

def read_file(path):
    p = (REPO / path).resolve()
    if not str(p).startswith(str(REPO)):
        return {"error": "path escapes repo"}
    try:
        return {"content": p.read_text()[:50_000]}
    except Exception as e:
        return {"error": str(e)}

def run_tool(name, args):
    if name == "list_files": return list_files(**args)
    if name == "read_file":  return read_file(**args)
    return {"error": f"unknown tool {name}"}

messages = [{"role": "user", "content": "Where is the rate-limit handling in this repo?"}]

for step in range(10):   # hard cap — see "what goes wrong" below
    resp = client.messages.create(
        model="claude-sonnet-4-6",
        max_tokens=1024,
        tools=TOOLS,
        messages=messages,
    )
    messages.append({"role": "assistant", "content": resp.content})
    if resp.stop_reason != "tool_use":
        # final answer
        print("\n--- final ---\n", resp.content[-1].text)
        break
    # run each tool call the model made
    tool_results = []
    for block in resp.content:
        if block.type == "tool_use":
            result = run_tool(block.name, block.input)
            tool_results.append({
                "type": "tool_result",
                "tool_use_id": block.id,
                "content": str(result),
            })
    messages.append({"role": "user", "content": tool_results})
```

Run it. Watch what the model picks.

**Break it on purpose (10 min)**

1. Remove the `range(10)` cap. Ask a vague question. Does the model loop forever, or does it stop? Put the cap back.
2. Drop the security check (`startswith(REPO)`). Ask it to read `/etc/passwd`. (Then put the check back. This is *exactly* why tool scoping matters.)
3. Make a tool description deliberately misleading. Does the model still use the tool correctly, or does it follow your description?

## Discuss (10 min)

Share with your cohort:

- The tool description you wrote — would a stranger understand when to use it?
- The worst failure mode you hit. Was it the model, or the way you described the tool?

## Done when

You have a working agent that uses tools to answer a real question about your repo, and you can name the three failure modes from memory.
