# L3.2 -- Production observability for agents

**Level:** 3 -- Proficient
**Time:** ~90 minutes
**Pair or solo:** Solo build; cohort review of dashboards.

## You'll be able to

- Trace an agent run end-to-end (model calls, tool calls, latencies, costs).
- Set up the three alerts every production agent needs.
- Distinguish a model regression from an infra regression by looking at the data.

## Read / watch (20 min)

- [Anthropic -- Observability for LLM applications](https://docs.claude.com/en/docs/build-with-claude/observability) (or your team's wiki on tracing if you have one).
- Pick one open-source LLM observability tool and read its docs for ten minutes: LangSmith, Langfuse, Phoenix, Helicone, or your existing APM (Datadog, Honeycomb).
- If your team has its own tracing setup, get a tour from a teammate who's used it.

## Do (60 min)

You'll instrument your L2.2 agent with structured tracing and stand up a minimal dashboard.

**Step 1 -- Pick a tool (5 min)**

Use what your team uses. If the team doesn't have one yet, default to **Langfuse self-hosted** or **OpenTelemetry → your existing observability backend**. Both are fine.

**Step 2 -- Instrument every model and tool call (25 min)**

Wrap your agent loop so that every step emits one structured event with at least:

```typescript
// emitted as JSON lines to stdout
{
  trace_id: "...",          // group calls in one run
  step: 3,
  kind: "model_call" | "tool_call",
  model: "claude-sonnet-4-6",
  tool: "read_file",        // if tool_call
  input_tokens: 1280,
  output_tokens: 245,
  cache_read_tokens: 1100,
  latency_ms: 1420,
  cost_usd: 0.0034,
  stop_reason: "tool_use",
  error: null,              // or the exception message
}
```

If you're using an SDK, hook into its existing tracing. If not, a simple wrapper works:

```typescript
async function withTrace<T>(kind: string, fields: Record<string, unknown>, fn: () => Promise<T>): Promise<T> {
  const t0 = Date.now();
  try {
    const result = await fn();
    emit({ kind, latency_ms: Date.now() - t0, error: null, ...fields });
    return result;
  } catch (e) {
    emit({ kind, latency_ms: Date.now() - t0, error: String(e), ...fields });
    throw e;
  }
}

function emit(event: Record<string, unknown>): void {
  process.stdout.write(JSON.stringify(event) + "\n");
}
```

Pipe output to a file: `npx ts-node your_agent.ts | tee traces.jsonl`

**Step 3 -- Build a one-screen dashboard (20 min)**

Doesn't matter if it's Grafana, a Notion table refreshed by a script, or a simple web dashboard. It must show:

- Requests per hour (last 24h).
- Median and p95 latency per step.
- Cost per request and total cost today.
- Error rate.
- Tool-use rate (% of runs that called any tool).

**Step 4 -- The three alerts (10 min)**

Define these (even if you wire them up later). Each one should answer a different question:

1. **"Is anything on fire?"** -- Error rate > 5% over 5 minutes.
2. **"Are we hemorrhaging money?"** -- Cost in the last hour > 2× the 7-day rolling average.
3. **"Did quality drop?"** -- Eval pass rate in CI dropped below threshold (this links to L3.4). Or, if you can't run evals live yet: tool-use rate dropped by >30% (often means the model is short-circuiting).

## Self-check before your check-in (5 min)

Make sure you can answer these from memory:

- Can you answer "which run cost most yesterday and why?" using only your emitted data?
- Name the three alerts and the threshold you'd set for each.
- How would you distinguish a model regression from an infra regression looking only at the dashboard?

## Check-in with your engineer lead (~15 min)

Book 15 minutes with your engineer lead. Bring:

- Your instrumented agent (or the trace JSONL output).
- A link or screenshot of your one-screen dashboard.
- Your three alerts written down (trigger + action).

The lead will ask you to walk through the dashboard for 5-7 minutes, then probe with questions like:

- "Triage one slow request live. Find the bottleneck."
- "Walk me through one alert. What page does it produce and at what threshold?"
- "Imagine the model regressed silently. How would your data show it?"

When the **Done when** bar below is met, the lead signs you off in your `status.md`.

## Done when

You can run a query that answers "which run cost the most yesterday, and why?" using the data your agent emits. You can also name the three alerts you'd want before going to production.
