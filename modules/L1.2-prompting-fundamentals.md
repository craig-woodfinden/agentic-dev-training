# L1.2 -- Prompting fundamentals

**Level:** 1 -- Novice
**Time:** ~90 minutes
**Pair or solo:** Solo for the exercise; share results with your pair at the end.

## You'll be able to

- Apply five concrete prompt patterns you'll reuse weekly.
- Tell when a prompt needs an example, a format spec, or a role -- rather than just rewording it.
- Spot the difference between a system-style instruction and a user message, and when each one matters.

## Read / watch (20 min)

- [GitHub Copilot -- Best practices for using Copilot Chat](https://docs.github.com/en/copilot/using-github-copilot/best-practices-for-using-github-copilot) -- read in full.
- [GitHub Copilot -- Prompting for Copilot Chat](https://docs.github.com/en/copilot/github-copilot-chat/about-github-copilot-chat) -- skim the section on writing effective prompts.
- [Prompting guide -- Zero-shot and few-shot prompting](https://www.promptingguide.ai/techniques/fewshot) -- read the few-shot section; skim the rest.

## Do (60 min)

You'll apply five patterns to the same real task -- extracting structured info from messy text -- and compare results. All five patterns are run in **GitHub Copilot Chat** in VS Code.

**Setup (5 min)**

Find a real piece of messy text from your work: a long Teams chat, a meeting note, a customer email, a JIRA comment. Anonymise it if needed. Paste it into a scratch file in VS Code as `INPUT`.

Your job for each pattern: extract a structured summary -- `{ "decisions": [...], "owners": [...], "open_questions": [...] }`.

Open the Copilot Chat panel. You will paste each prompt directly into the chat. Between patterns, start a fresh conversation (use the "New conversation" button) so earlier messages do not bleed into the next result.

**Pattern 1 -- Bare instruction (10 min)**

Send a one-line prompt: `"Summarise the decisions, owners, and open questions from this text: <INPUT>"`. Save the output.

**Pattern 2 -- Instruction + format spec (10 min)**

Add an explicit format to the same prompt: `"Return strict JSON with keys decisions (list of strings), owners (list of strings), open_questions (list of strings)."`. Run again.

**Pattern 3 -- Add an example (few-shot) (10 min)**

Before your real input, prepend a short worked example showing one paragraph of input and the exact JSON output you want. Then add your real input. Run again.

**Pattern 4 -- Role instruction (10 min)**

Open the prompt with a standing instruction that sets the role: `"You are an analyst who extracts structured information from meeting notes. Be precise and do not invent information that is not present."` Then add the task and your input. Run again.

This mirrors what a `system` prompt does in the API -- you are front-loading the instructions that frame every answer. In Copilot Chat you do this at the start of your message rather than in a separate field.

**Pattern 5 -- Prefill the answer (10 min)**

Start your message with the opening of the JSON structure you want, then ask Copilot to complete it. For example:

`"Complete the following JSON using only information from the text below. Do not add keys. Text: <INPUT>"`

Then on the next line, begin:

`{"decisions": [`

Paste both together as your single chat message. Copilot will tend to continue the structure you started rather than inventing a new format. This is the Copilot Chat equivalent of the API prefill technique.

**Compare (5 min)**

Open all five outputs side by side (in separate editor tabs or a doc). Which version produced the cleanest, most parseable result? Which would you trust to drop into a script? Save the winner as `prompt_v1.txt` -- that's your first reusable prompt.

## Self-check (5 min)

Make sure you can answer these from memory:

- Name the five patterns.
- Which pattern made the biggest difference for your task, and which made surprisingly little?
- When would adding few-shot examples *hurt* a prompt (think: tokens, biasing)?

## Done when

You have one prompt saved as a reusable file, and you can name the five patterns from memory.
