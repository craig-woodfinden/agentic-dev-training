# L1.3 — Daily Claude Code workflow

**Level:** 1 — Novice
**Time:** ~75 minutes
**Pair or solo:** Solo, then share your CLAUDE.md and one slash command with your pair.

## You'll be able to

- Drive Claude Code (or the team's chosen agentic CLI) end-to-end for a small, real task.
- Write a useful `CLAUDE.md` for one of your repos.
- Create one personal slash command that you'll keep using.

## Read / watch (15 min)

- [Claude Code — Quickstart](https://docs.claude.com/en/docs/claude-code/quickstart)
- [Claude Code — CLAUDE.md files](https://docs.claude.com/en/docs/claude-code/memory)
- [Claude Code — Slash commands](https://docs.claude.com/en/docs/claude-code/slash-commands)

If your team uses Cursor or another agentic CLI instead, swap in those equivalents — most of this module still applies.

## Do (50 min)

You'll set up Claude Code on a repo you know, then use it to ship a small change.

**Setup (10 min)**

1. Install Claude Code: follow the install docs for your OS.
2. Run `claude` in a repo you actually work in.
3. Authenticate.

**Write a CLAUDE.md (15 min)**

In the root of your chosen repo, create or update `CLAUDE.md`. Write the file as if onboarding a new engineer in five paragraphs:

- What this repo is for (1 paragraph).
- How to run it locally — exact commands (1 paragraph).
- The two or three patterns that are easy to get wrong (1 paragraph).
- Where the tests live and how to run a single test (1 paragraph).
- Any do-not-touch areas, and why (1 paragraph).

Resist the urge to make it long. The whole file should fit on one screen.

**Ship a small change (15 min)**

Pick a small, real chore you've been putting off in this repo. A rename, a missing test, a docstring sweep, a small bug. Drive Claude Code to do it. Some prompts to try:

- "Read CLAUDE.md, then…"
- "Show me the smallest possible diff that would do X. Don't write it yet — describe it first."
- "Run the tests for just the file you changed."

Commit the work in a branch. Don't merge yet.

**Write one slash command (10 min)**

Pick a thing you find yourself typing over and over. Make a slash command for it. Save it in your user-level commands so you keep it across repos. Some good first commands:

- `/test-this` — runs the test file for whatever you're looking at.
- `/branch-summary` — summarises the diff against `main` for a PR description.
- `/explain-this` — gives a 5-line explanation of the file or function you're on.

## Discuss (10 min)

Share with your pair:

- Your `CLAUDE.md`. What did they think was missing?
- Your slash command. Did they steal it?
- Anything Claude Code did *wrong* on your small change, and how you caught it.

## Done when

You have:
- A `CLAUDE.md` in one real repo
- One slash command you've saved and used at least twice
- A branch with an AI-assisted commit that you'd be willing to put up for review
