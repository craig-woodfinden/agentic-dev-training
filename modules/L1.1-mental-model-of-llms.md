# L1.1 — A working mental model of LLMs

**Level:** 1 — Novice
**Time:** ~75 minutes
**Pair or solo:** Either works. Pairing is encouraged.

## You'll be able to

- Explain tokens, context window, and sampling temperature in plain English in under two minutes.
- Predict the kinds of questions where a model is most likely to make something up.
- Read an API response and tell someone what it actually cost.

## Read / watch (15 min)

- [Anthropic — Models overview](https://docs.claude.com/en/docs/about-claude/models/overview) — skim the table; note which models exist and roughly what they're for.
- [Anthropic — Prompt engineering intro](https://docs.claude.com/en/docs/build-with-claude/prompt-engineering/overview) — read the first two sections only.

If you have ten extra minutes, watch any one of Anthropic's short "how Claude works" intro videos on YouTube. Don't go down a rabbit hole — come back.

## Do (50 min)

You'll write a tiny script, run it three ways, and notice what changes.

**Setup (5 min)**

```bash
pip install anthropic
export ANTHROPIC_API_KEY="..."   # ask your manager or check the team password manager
```

**Script (5 min)**

Save this as `hello_claude.py`:

```python
import anthropic
client = anthropic.Anthropic()

resp = client.messages.create(
    model="claude-haiku-4-5",
    max_tokens=200,
    temperature=0.7,
    messages=[
        {"role": "user", "content": "Write a four-line poem about debugging at 2am."}
    ],
)
print(resp.content[0].text)
print(f"\n--- in: {resp.usage.input_tokens} tokens, out: {resp.usage.output_tokens} tokens")
```

Run it. Read the output. Look at the token counts.

**Experiment (30 min)**

Run the script three more times, changing one thing each time. After each run, jot down one line about what you noticed.

1. Set `temperature=0.0` and run twice. Are the outputs identical? Why?
2. Set `temperature=1.0` and run twice. What changed?
3. Drop `max_tokens` to `20`. What happens to the output — is it gracefully shorter, or cut off mid-word?
4. Change the prompt to ask Claude to "name three real Anthropic papers published in 2024 with arXiv links." Run it. **Do not trust the answer.** Try to verify one of the links. What did you learn about hallucination?

**Cost it (10 min)**

Look up the per-token price for `claude-haiku-4-5` in the [pricing docs](https://docs.claude.com/en/docs/about-claude/pricing) or on the team wiki. Compute the cost of your most expensive run above. Write the number down.

## Self-check before your check-in (5 min)

Make sure you can answer these from memory before you book time:

- Explain `temperature` and `max_tokens` in one sentence each.
- Roughly, what did your most expensive run cost?
- Where in your day-to-day work would you most expect hallucination, and what would you do about it?

If you can't, re-read the relevant section. The check-in is short — it works best when you arrive ready.

## Check-in with your engineer lead (~15 min)

Book 15 minutes via Slack DM. Bring:

- Your `hello_claude.py` with the modifications you made.
- The token counts and cost from your most expensive run, written down.
- One example of hallucination you produced on purpose, with the prompt.

The lead will ask you to walk through what you did for 5–7 minutes, then probe with questions like:

- "Show me the token counts on this run — what does each one cost?"
- "Walk me through what changed when you raised temperature."
- "Where did the model invent something, and what in the prompt invited it?"

When the **Done when** bar below is met, the lead signs you off in your `status.md`.

## Done when

You can show someone a script you wrote, the token counts in the response, and tell them what the call cost. You can also describe — in one sentence each — what `temperature` and `max_tokens` actually do.
