# L1.1 -- A working mental model of LLMs

**Level:** 1 -- Novice
**Time:** ~75 minutes
**Pair or solo:** Either works. Pairing is encouraged.

## You'll be able to

- Explain tokens, context window, and sampling temperature in plain English in under two minutes.
- Predict the kinds of questions where a model is most likely to make something up.
- Describe the difference in behavior between creative, factual, and structured-output prompts.

## Read / watch (15 min)

- [GitHub Copilot -- Overview](https://docs.github.com/en/copilot/about-github-copilot/what-is-github-copilot) -- read the intro; note what Copilot is and isn't.
- [GitHub Copilot -- Getting started with Copilot Chat](https://docs.github.com/en/copilot/github-copilot-chat/about-github-copilot-chat) -- skim.
- [3Blue1Brown -- But what is a GPT? (YouTube)](https://www.youtube.com/watch?v=wjZofJX0v4M) -- watch the first 10 minutes only. This is the clearest visual explanation of tokens and prediction available. Don't go down a rabbit hole -- come back.

## Do (50 min)

You'll ask Copilot Chat three types of questions, run one temperature-equivalent experiment, and manufacture a hallucination on purpose.

No API key required. You need VS Code with the GitHub Copilot extension installed and active.

**Setup (5 min)**

Open VS Code. Confirm the Copilot Chat panel is available (the chat icon in the Activity Bar, or `Ctrl+Alt+I` / `Cmd+Alt+I`). If it isn't installed, follow the [Copilot quickstart](https://docs.github.com/en/copilot/quickstart) before continuing.

Open the Chat panel and make sure you can send a message.

**Three prompt types (15 min)**

Send each of these prompts in a fresh Copilot Chat session. After each, jot one line about what the response was like.

1. Creative: `"Write a four-line poem about debugging at 2am."`
2. Factual recall: `"What does the HTTP 429 status code mean and when would a server return it?"`
3. Structured output: `"Return strict JSON with keys title, author, and year for the book 'The Pragmatic Programmer'."`

Notice what changed: tone, confidence, formatting, length. You're observing how the same model behaves differently depending on the shape of the task.

**Temperature-equivalent experiment (15 min)**

Copilot Chat does not expose a temperature slider, but the underlying model samples probabilistically. Ask the same open-ended prompt three times in separate sessions and compare the outputs:

`"Give me three unconventional names for a TypeScript utility function that retries a failed async call."`

Are the answers identical? Partially overlapping? Completely different? Write down one observation. This is temperature in action -- you're watching it, not tuning it. (When you reach L2 and use the API directly, you will tune it.)

**Manufacture a hallucination on purpose (15 min)**

Ask Copilot Chat:

`"Name two real papers about transformer attention mechanisms published on arXiv in 2023. Include the arXiv links."`

Do not trust the answer. Open each link in a browser. What did you find? If a link 404s, or the paper title does not match, or the authors are wrong -- that is a hallucination. Write down exactly what was wrong.

This is not a Copilot failure to mock; it is the expected behavior of a probabilistic text predictor when asked to recall specific identifiers. You need to internalize this before relying on any model for research or citations.

## Self-check (5 min)

Make sure you can answer these from memory:

- Explain `temperature` and `context window` in one sentence each.
- Which of your three prompt types produced the least reliable output, and why?
- Where in your day-to-day work would you most expect hallucination, and what would you do about it?

If you can't, re-read the relevant section.

## Done when

You can describe -- in one sentence each -- what `temperature` and `context window` actually mean. You can show someone a hallucination you manufactured on purpose, explain exactly what was wrong, and say what kind of prompt is most likely to produce one.
